# Wireshark Network Traffic Analysis

## Goal
Capture and analyze network traffic to understand common protocols and identify potential security risks.

## Tools Used
- Wireshark
- Windows / Linux
- Basic networking knowledge (DNS, HTTP, TCP)

## What I Did
1. Captured live network traffic in Wireshark.
2. Applied protocol filters to isolate DNS, HTTP, and TCP traffic.
3. Identified examples of unencrypted traffic and documented why it can be risky.
4. Summarized findings and suggested basic mitigations.

## Key Skills Demonstrated
- Packet inspection and filtering
- Understanding of DNS, HTTP, and TCP behavior
- Identifying unencrypted communication risks
- Writing clear security notes for non-technical audiences

## Files
- `filters.md` — filters used + what each one shows
- `findings.md` — observations + security notes
- `screenshots/` — screenshots of traffic examples (no sensitive info)

## Notes
All captures were taken from my own device/network in a learning environment. No private or unauthorized traffic was collected.
